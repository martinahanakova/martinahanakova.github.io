V zadaní 3 som vytvorila prezentáciu v XML. Jej schému som definovala v Ralax NG kompaktnej forme a pomocou XSL tranformácií som ju transformovala do XHTML a PDF formátu.

Opis a účel navrhnutých elementov

  • presentation

    • Koreňový element, ktorý obsahuje všetky nasldujúce elementy.

  • slide

    • Element slide obsahuje všetky elementy, ktoré popisujú obsah na slajde. Môže byť dvoch typov, titulný alebo obsahový.

  • titleSlide

    • Titulný slajd obsahuje nadpis, podnapis a autorov, pričom pre každého autora je zvlášť definované meno a priezvisko.

    • title
    • subtitle
    • authors

      • firstname
      • lastname

  • header

    • Element header obsahuje nadpis a podnadpis slajdu. Tento element je nepovinný a môže obsahovať aj len nadpis alebo len podnadpis.

    • title
    • subtitle

  • content

    • Element content obsahuje všetok obsah slajdu. Obsahuje nižšie definované elementy, v ľubovoľnom počte a poradí, aby mal autor prezentácie možnosť si čo najviac prispôsobiť rozloženie prvkov na slajde.

    • paragraph

      • Element paragraph obsahuje všetky nižšie uvedené elementy taktiež v ľubovoľnom počte a poradí.

      • link

        • Hypertextový odkaz.

      • cite

        • Element označuje odkaz na niektorý zo zdrojov bibliografie.

      • highlight

        • Element slúži na zvýraznenie textu. Text môže autor zvýrazniť buď tučným písmom, kurzívou alebo podčiarknutím.

      • code

        • Code slúži na zvýraznenie časti kódu odlišným fontom písma. Taktiež sú zachované všetky medzery a "nové riadky".

      • list

        • Element lists slúži na definovanie zoznamov, či už neusporiadaných alebo číselných.

      • image

        • Slúži na vloženie obrázka. Pri každom obrázku je potrebné definovať jeho zdrojovú adresu, veľkoť a centrovanie.

    • image
    • code
    • list

      • Element list obsahuje jednotlivé položky zoznamu označené ako item. Každá položka môže obsahovať nižšie uvedené elementy, ktoré slúžia zvýraznenie alebo doplnenie textu v zozname. Zoznam tiež môže obsahovať ďalšie vnorené zoznamy.

      • link
      • cite
      • highlight
      • code
      • list

    • columns

      • Umožňuje rozdeliť slajd do dvoch stĺpcov. Každý stĺpec môže obsahovať element paragraph alebo image. Týchto elementov môže obsahovať aj viac.

  • bibliography

    • Slúži na definovanie použitých zdrojov a literatúry.

    • abbrev

      • Tento element slúži ako jedinečný identifikátor zdroja. Pomocou neho sa dá v texte na daný zdroj odkazovať.

    • authors

      • Definuje zoznam autorov zdroja.

    • title
    • year
    • link

XSL transformácie

Na transformáciu xml prezentácie do formátov XHTML a PDF som použila xsl transformácie. Pre každý elemnent som si vytvorila samostaný "template", podľa ktorého príslušný element transformujem. Pre konverziu xml dokumentu do xhtml a pdf som použila skoro rovnaké xsl transformácie. Rozdiel je v pridaní fo:block pri transoformácii do pdf. Pri použití týchto bolokov som tiež musela definovať ich "správanie". To, čo mi v xhtml zabezpečovali rôzne tagy, som teraz musela nastaviť pre fo:blocky pomocou ich atrinútov. Pri niektorých transformáciách som využila aj jazyk XPath, napríklad na zistenie pozície elemntu, čo využívam pri číslovaní slajdov.

Parametre

Po vytvorení všetkých transformácií som ešte naformátovala obsah výsdných dokumentov pomocou css alebo atribútov fo:blockov. S tým súvisí aj parametrizácia a teda možnosť nastaviť si niektoré "atribúty" výsledného dokumentu. Pri tvorbe prezentácie umožňujem autorovi nastaviť si vlastnú veľkosť písma pre nadpisy, podnadpisy a paragrafy. Veľkosť písma si môže nastaviť zvlášť pre titulný slajd a potom pre zvyšok prezentácie. Umožňujem mu tiež meniť farbu písma a pozadia. Farbu pozadia môže meniť zvlášť pre jednotlivé časti (header, content a footer). Pre každú časť je možné miesto farby pozadia nastaviť podkladový obrázok. V prípade, že je v parametroch uvedená aj podkladová farba, aj obrázok, zobrazí sa obrázok. Okrem toho ešte umožňujem nastaviť si vlastný font písma a veľkosť obrázka (len pre pdf, pre xhtml je definovaná pomocou atribútov). Parametre sa zvlášť nastavujú pre pdf a zvlásť pre xhtml formát. Robím tak z dôvodu odlišného zobrazovania veľkosti (čo je pre xhtml akurát, je pre pdf príliš veľké).
